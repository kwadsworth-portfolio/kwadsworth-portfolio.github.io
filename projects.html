Project 1: AWS Cloud Security Fundamentals

Cloud security initiative focused on identity, access control, network segmentation and auditability.

Objective

To design and deploy a secure AWS environment, demonstrating how cloud security principles can be applied to reduce risk and support governance and compliance requirements.

What I Implemented

  • Secure AWS account setup with foundational security controls.
  • IAM users, roles and policies aligned to least-privilege principles.
  • VPC design with public and private subnet separation.
  • Network access controls using security groups.
  • Centralised logging and monitoring to support investigation.

Approach & Outcomes

  • Applied a structured delivery approach: scope, risk identification, control implementation.
  • Documented security decisions and residual risks.
  • Produced artefacts suitable for walkthroughs and interviews.

Project 2: SIEM Visibility & Incident Detection

Security monitoring and detection initiative focused on event visibility and response workflows.

Objective

To establish security visibility through log collection and demonstrate how raw events become actionable alerts within a SIEM.

What I Implemented

  • Connected systems to a SIEM for log ingestion.
  • Created detection rules for suspicious authentication activity.
  • Built dashboards to highlight trends and anomalies.
  • Performed basic alert triage and investigation.

Approach & Outcomes

  • Mapped detections to common attack behaviours.
  • Demonstrated the link between technical alerts and business risk.
  • Documented findings and limitations of detection coverage.

Project 3: Cyber GRC & Risk Management (Mini-ISMS)

Governance, risk and compliance initiative aligned to ISO 27001 principles.

Objective

To design a lightweight Information Security Management System (ISMS) demonstrating how governance and risk management support effective security decision-making.

What I Implemented

  • Asset identification and data classification.
  • Risk register capturing threats, likelihood and impact.
  • Selection of appropriate security controls.
  • Drafted core security policies and procedures.

Approach & Outcomes

  • Ensured traceability between risks, controls and governance objectives.
  • Applied project discipline to structure documentation and reviews.
  • Created a practical foundation suitable for a small organisation.
