Project 1: AWS Cloud Security Fundamentals Lab

Cybersecurity and cloud security initiative designed around real-world use cases, focusing on identity, access control, network segmentation and auditability.

Objective

To design and deploy a secure AWS environment and demonstrate how identity management, access control, network separation and logging can be combined to reduce risk and support governance requirements.

What I Implemented

  • Created an AWS environment with security guardrails and account protections enabled.
  • Configured IAM users, roles and policies following least-privilege principles.
  • Designed a VPC with public and private subnets to separate internet-facing and internal resources.
  • Restricted access using security groups and network controls aligned to defined use cases.
  • Enabled CloudTrail and basic monitoring to support auditability and incident investigation.

Approach & Outcomes

  • Applied a structured, project-style approach: scoped the environment, identified risks, then implemented controls.
  • Documented security decisions, assumptions and residual risks for future review.
  • Produced architecture diagrams and configuration evidence suitable for walkthroughs or interviews.

Project 2: SIEM Visibility & Detection Lab

Security monitoring and detection initiative based on real-world event and alert use cases, focused on visibility, investigation and informed response.

Objective

To establish security visibility using a SIEM platform and demonstrate how events move from raw logs to meaningful alerts that can support an incident response workflow.

What I Implemented

  • Onboarded test systems to a SIEM to collect authentication and system activity logs.
  • Created detection rules for suspicious activity such as repeated failed logins and unusual process execution.
  • Built simple dashboards to visualise trends and highlight notable events.
  • Defined a basic workflow from alert triage through to investigation and response.

Approach & Outcomes

  • Mapped detections to common attack behaviours to understand coverage and gaps.
  • Captured example alerts and investigations to demonstrate practical understanding of SIEM usage.
  • Used the lab to connect technical events back to business impact and risk.

Project 3: Cyber GRC & Risk Management Mini-ISMS

Governance, risk and compliance initiative designed to reflect real-world organisational risk contexts, control selection and assurance requirements.

Objective

To design a lightweight Information Security Management System (ISMS) for a fictional organisation and demonstrate how governance, risk management and security controls can be aligned with business objectives.

What I Implemented

  • Defined a simple asset inventory and data classification approach.
  • Built a risk register capturing threats, likelihood, impact and treatment decisions.
  • Drafted core information security policies (e.g. access control, incident management, acceptable use).
  • Mapped identified risks to appropriate security controls and documented rationale.

Approach & Outcomes

  • Applied project management discipline to structure the ISMS: scope, requirements, artefacts and review.
  • Emphasised traceability from business risks through to chosen technical and procedural controls.
  • Produced documentation suitable as a starting point for a small-organisation ISMS.
